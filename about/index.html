About / 為什麼做這個站

這個 blog 想長成一本 KAI 的 infra notebook

我想做的不是雜亂文章倉庫，而是一個更像 notebook 的個人技術站。主線是 KAI K8s之路：先用生活例子把直覺搭起來，再回到 Kubernetes 的資源邊界、controller 行為與 production 排查。副線則收進 GitOps、observability、AI infra，以及那些值得記下來的事故筆記。

Writing bias / 寫作偏好

  • 每章先用一個生活例子建立直覺
  • 再把抽象概念放回 Kubernetes 真正的技術邊界
  • 偏好真實現場，不想只寫擺樣子的 demo
  • 寫到自己未來回來看也還會有用

主線怎麼寫

先講好懂的直覺，再回到專業邊界

系列會先把 Pod、Deployment、Service、Probe、Ingress 這些關鍵觀念講清楚：每章都有生活例子，但最後一定回到 controller、traffic、rollout、GitOps、observability 與 incident handling。

英文怎麼做

繁中是主稿，英文只是 mirror

系列內容以繁中筆記為準，英文版保留技術對照與對外分享用途；支線內容不強迫一比一翻譯。

支線會放什麼

排查筆記、操作守則、概念模型

除了主線系列，也會保留 probe / rollout / GitOps 類的現場經驗、誤判模式，以及值得記下來的 production 事故筆記。

Working principles

我希望每篇文章至少做到這三件事

  • 回答一個真實會遇到的問題
  • 用一個生活例子讓人先理解，但不犧牲技術精準度
  • 給出我自己會採用的默認做法
  • 寫到別人能拿去用，而不是只能看完點頭
